# Contributing to Sajda App

Thank you for your interest in contributing to Sajda App.

## General Guidelines

- Keep changes focused and easy to review
- Do not mix unrelated changes in a single commit or pull request
- Maintain UI consistency with Sajda App's visual direction
- Avoid destructive changes unless they have been discussed first

## Before Opening a Pull Request

Please make sure that:

- the app still builds
- the main user flows are not broken
- UI changes remain clean on the relevant screens
- screenshots are included if the contribution changes the interface
- no local files or build artifacts are committed by mistake

## Good Contribution Areas

- bug fixes
- UI polish
- prayer and adhan reliability
- Qur'an reading experience
- audio workflow
- documentation

## Pull Request Notes

A good pull request should clearly explain:

- what was changed
- why the change was needed
- which areas may be affected
- how the change can be tested

## Discuss First

For larger changes such as architecture updates, adhan system changes, database changes, or major UI redesigns, please discuss the idea first before implementing it fully.
